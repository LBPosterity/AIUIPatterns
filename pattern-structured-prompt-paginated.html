Structured Prompt Paginated

Overview

Similar to the "Structured Prompt" the paginated prompt provides clear guidance for the prompts structure with a strong focus on each section as they're individually presented.

Whilst the users progress is indicated it's generally more effective to limit the prompt segmentation to 3-6 steps.

User Archetype

A pattern generally used for contexts like onboarding, this approach is optimised for first time or novice users as it focuses their attention on one section at a time.

Each step also provides the opportunity to present help information bespoke to each individual section.

Considerations

Commonly all steps in a paginated workflow are required, so consider carefully the content required for a quality prompt & if it can be compartmentalised reliably.

While paginated workflows focus users, they can also take longer to complete. Consider the trade off between the longer compeletion time & guiding users to higher quality prompts.

Another context dependant consideration is the inability to cross reference a previous sections content. As these sections would be combined into a single prompt on submission, consider if a user would benefit form cross referencing one sections content as they author another.
